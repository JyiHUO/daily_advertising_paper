## 三、每日一个知识点

- **主题：** 上下文 Bandit（Contextual Bandit）
- **是什么：** 上下文 Bandit 是一种在线决策框架：每一轮系统看到一个描述当前情境的「上下文」（比如用户画像和页面信息），然后从多个可选动作（比如展示哪条广告）中选一个执行，执行后只能观察到所选动作的反馈（点击或不点击），看不到其他动作的反馈。系统需要在「利用已知好选项」和「探索可能更好但不确定的选项」之间持续权衡。它比普通多臂老虎机更贴近真实业务，因为决策会根据当下情境变化。
- **为什么值得学：** 今天候选论文中排名最高的一篇正是研究线性上下文 Bandit 如何在恶劣噪声下保持鲁棒性。广告系统中每次请求要从候选广告中快速选一个展示，本质上就是一个上下文 Bandit 问题，理解它有助于读懂广告出价、冷启动和探索策略相关论文。
- **例子：** 假设你运营一个电商广告位：用户 A 是年轻女性、正在浏览运动鞋页面（这就是上下文），系统需要从 50 条广告中选一条展示。如果总选历史点击率最高的广告（利用），可能错过新上的优质广告；如果偶尔试试没展示过的广告（探索），就有机会发现更好的匹配。上下文 Bandit 算法就是帮你在每一次请求中自动平衡这个取舍。

## 四、分分类知识点速览

- **cs.IR：召回和排序的分工**
  - 是什么：信息检索里常把系统拆成召回和排序两层。召回负责从海量候选里先快速找出少量可能相关的结果，排序再用更重的模型精细比较。
  - 为什么值得知道：广告系统也常是同样结构：先找可能能投的广告，再做精排和出价决策。理解这层分工，能更快判断论文改进发生在系统哪一段。
- **cs.AI：问题建模比模型复杂更重要**
  - 是什么：AI 论文里很多提升并不只是换一个更大的模型，而是先把目标、约束、反馈信号和决策边界定义清楚。问题建模对了，后面的学习和推理才有意义。
  - 为什么值得知道：广告业务经常不是单目标优化，而是同时平衡点击、转化、预算和体验，所以建模方式往往比单点算法更关键。
- **cs.LG：泛化能力不等于训练集效果**
  - 是什么：机器学习里一个模型在训练或验证集上表现好，不代表换场景后还有效。泛化讨论的是模型能不能把规律带到新流量、新人群和新分布中。
  - 为什么值得知道：广告环境变化快，节假日、活动、市场切换都会让分布漂移，所以看论文时要特别关注它如何处理泛化与稳健性。
- **cs.CL：语义表示决定语言理解上限**
  - 是什么：自然语言处理里，很多任务表面不同，但底层都依赖文本表示是否准确。好的语义表示能把词面不同但意思接近的内容拉近，也能把看似相似但意图不同的内容区分开。
  - 为什么值得知道：广告检索、创意理解、query 意图识别都强依赖语义表示，所以 cs.CL 的很多工作可以迁到广告理解层。
- **cs.MA：多智能体的关键是协同而不是单体最优**
  - 是什么：多智能体系统里，每个体单独做得好，并不代表整体效果最好。真正难的是信息怎么共享、目标怎么对齐、冲突怎么协调。
  - 为什么值得知道：广告投放里预算、创意、渠道、策略代理如果分别优化，可能互相打架；多智能体视角能帮助理解系统级协调问题。

## 七、重点论文精读

### 1. Robust and Computationally Efficient Linear Contextual Bandits under Adversarial Corruption and Heavy-Tailed Noise
- **背景：** 论文研究线性上下文 bandit 在同时存在对抗式污染和重尾噪声时，如何既稳健又高效地做决策。已有方法要么只抗一种问题，要么要求更强的有限方差假设，要么每轮都要在全部历史数据上解一个优化问题，计算代价会随时间增长。本文值得看，是因为它把更弱的噪声假设、对污染的鲁棒性和在线高效更新放到了一起，而且摘要与正文都明确提到广告系统里的点击欺诈就是典型动机之一。
- **核心技术点：**
  - **点 1：问题建模更贴近脏数据**
    - 技术细节：论文定义每轮先看到候选动作集合和对应上下文向量，选择一个动作后，真实随机回报等于上下文和未知参数的线性内积再加上噪声；最终观测到的却是再被额外加上污染项后的回报。它把总污染量定义为所有轮次污染绝对值之和，并只要求噪声满足较弱的有界一加epsilon阶矩，而不是必须有有限二阶矩。目标仍是最小化累积后悔，也就是和每轮最优动作的线性期望回报相比少拿了多少。
    - 通俗讲解：可以把它理解成：系统本来想根据用户和物料特征学出谁更可能带来收益，但拿到的反馈里既可能混入恶意篡改，也可能出现极端大值。论文先把这两个现实问题一起写进模型里，再讨论算法怎么扛住它们。
    - 例子：比如广告系统每次给 20 个候选创意，系统选一个展示。真实转化价值本来由用户特征和创意特征共同决定，但某些轮次点击被作弊流量抬高，另一些轮次又因为少数高客单用户导致收益特别极端。论文的模型就是把这两种情况同时纳入反馈生成过程。
  - **点 2：自适应缩放的鲁棒损失**
    - 技术细节：算法核心不是直接拿观测回报做普通平方误差，而是先把预测误差除以一个自适应缩放量，再送进 Huber 损失。这个缩放量由几部分取最大值：噪声规模下界、一个很小的数、与当前动作不确定性相关的项、以及与总污染预算和当前动作椭圆范数相关的污染项。直观上，当某轮可能更脏或更不确定时，缩放量就变大，归一化后的误差变小，损失梯度会被压住；而 Huber 损失在误差小时像平方损失，误差太大时改成近似线性增长，避免极端样本把参数拉飞。论文还额外设置了一个随当前不确定性变化的阈值，用来控制什么时候从平方区切到线性区。
    - 通俗讲解：这相当于给每条反馈先做风险评估：越像异常值、越像被污染、或者当前本来就越不确定，这条样本的话语权就越小。不是完全丢掉异常反馈，而是让它别主导模型更新。
    - 例子：假设某次广告展示的观测收益突然高得离谱，普通最小二乘会强烈相信这条样本，参数可能朝错误方向大跳。这里会先把这条误差按缩放量缩小，再用鲁棒损失裁掉过大的梯度，于是这次异常收益只会带来有限影响，模型不会因为一次假高回报就把类似广告都推到前面。
  - **点 3：OMD单遍高效更新**
    - 技术细节：论文没有像之前方法那样每轮把历史样本全部拿出来重新做一次凸优化，而是用在线镜像下降做递推更新。它维护一个数据相关矩阵，这个矩阵按当前样本的上下文外积累加，但权重是缩放量的倒数平方，因此越可疑的样本进入矩阵的权重越小。每轮先按当前梯度和这个矩阵的逆做一步中间更新，再把结果投影回参数约束集合；论文给出了两步形式：先梯度步，再按该矩阵诱导的范数做投影。文中强调，相比需要随时间增长的迭代优化，这个做法在关注轮次和总时长依赖时可视为每轮 O(1) 的在线更新；不过从维度角度看，仍需维护矩阵逆和投影，实际实现还要承担和特征维度有关的代价，这一点需要读者注意。
    - 通俗讲解：你可以把它看成从'每次重训全模型'改成'每来一条数据就快速修正一次'。这样更像工业实时系统，适合流式反馈、低延迟决策和大规模线上更新。
    - 例子：如果广告平台每秒都有新曝光反馈，旧方法可能需要回看全部历史并重复求解，很难卡住实时 SLA。本文方法只基于当前反馈、当前梯度和维护好的矩阵做一步修正，像在线调参一样持续更新，所以更适合边投放边学习。
  - **点 4：UCB决策与后悔保证**
    - 技术细节：动作选择部分采用标准的上置信界逻辑：对每个候选动作，用当前参数估计的期望回报加上一个与不确定性成正比的探索奖励，然后选总分最高的动作。不确定性由动作在当前矩阵逆下的范数给出，置信半径由理论分析给出。结果上，已知噪声矩界和污染预算时，后悔上界由两部分相加组成：一部分和噪声矩规模有关，另一部分和污染总量有关；当 epsilon 等于 1 时，能恢复有限方差场景的已有结果；当没有污染时，能匹配重尾线性上下文 bandit 的最好已知量级。论文还给出预算或噪声规模未知时的版本，用上界替代后仍能得到次线性后悔，但界会更保守。
    - 通俗讲解：策略层面上，它仍然是'高估不确定的潜力，再去探索'，只是这个不确定性估计和参数学习都做了鲁棒化。这样既不容易被脏数据骗偏，也不会因为太保守而完全不探索。
    - 例子：在广告排序里，一个新创意的历史很少，不确定性高，UCB 会给它探索加分；但如果它的少量反馈里夹杂疑似作弊高点击，鲁棒学习会减少这些样本对参数和置信区间的误导。结果是系统还会探索新创意，但不至于因为几次异常点击就大规模放量。
  - **点 5：实验显示更快更稳**
    - 技术细节：实验部分比较了本文方法和 Yu 等 2025 的方法。论文构造了有污染和无污染两种场景，并分别测试 epsilon 等于 1 与小于 1 的重尾噪声。在 epsilon 等于 1 时，本文方法后悔与对比方法相近，但运行时间低几个数量级；在 epsilon 小于 1 时，对比方法后悔接近线性增长，而本文仍表现出次线性后悔。需要说明的是，当前摘录里没有完整图表数值，因此具体领先幅度只能做定性描述，不能精确报数。
    - 通俗讲解：实验传达的信息很直接：在原来假设成立的情况下，它没明显牺牲效果，但更快；在原来假设不成立时，它更稳，不容易直接失效。
    - 例子：如果广告反馈噪声只是普通大一些，旧方法和新方法可能都能跑；但一旦出现更极端的价值波动，比如少量超高 GMV 用户订单，新方法还能保持学习，旧方法可能越来越偏，累计损失快速扩大。
- **对广告的启发：** 最适合层级：在线探索层，尤其适合广告创意选择、推荐曝光排序中的探索分、冷启动流量分配、以及实时出价中的价值估计更新；价值：最大启发是把'鲁棒反馈建模'和'实时单遍更新'结合起来。广告系统常见的点击欺诈、延迟归因误差、极端转化价值都属于污染或重尾反馈来源，这篇方法提示我们可以在 bandit 或轻量在线学习层对单条反馈做自适应降权，而不是只在离线清洗数据。对于需要毫秒级决策的系统，它比每轮重训更有工程可迁移性。；风险：第一，论文本身不是广告论文，迁移到 CTR、CVR、多目标出价时需要重新适配目标和反馈定义。第二，理论是在线性回报模型下建立的，若广告场景强非线性，直接套用效果不确定。第三，算法里一些量如噪声上界、污染预算或其替代上界在业务中未必容易设定，设得过大可能过于保守、探索不足。第四，文中把每轮复杂度称为相对常数级主要是相对时间轮次而言，实际仍依赖特征维度和矩阵维护成本，高维广告特征需要再做近似或低秩化工程设计。

### 2. Embedding-Aware Feature Discovery: Bridging Latent Representations and Interpretable Features in Event Sequences
- **背景：** 论文关注事件序列建模里的一个现实矛盾：研究里越来越多用预训练 embedding 和大模型，但生产系统依然大量依赖人工统计特征，因为这些特征更可解释、在标注少时更稳、延迟也更可控。问题在于两套体系通常各做各的，导致 embedding 里已经有的信息没人能解释，embedding 没学到的信息也没人系统性补上。作者因此提出 EAFD，想把'embedding 表征'和'可解释特征工程'连成一个闭环，这对任何依赖用户行为序列的工业系统都值得看，尤其适合广告推荐做强特征补盲。
- **核心技术点：**
  - **点 1：双分数找特征**
    - 技术细节：EAFD 先固定一个预训练事件序列编码器，把每个用户序列映射成 embedding，编码器参数在特征发现阶段不更新。然后让 LLM 从原始事件序列里提出候选特征，每个特征都要经过两个分数评估：一是对齐分数，用一个小型梯度提升模型判断该特征能否解释 embedding 中已有信息，文中说用重建一致性或相关性来衡量，并给出 R2 作为主要指标；二是下游效用分数，比较只用 embedding 的模型损失和同时用 embedding 加该特征的联合模型损失，若加入特征后损失下降，则该特征提供了 embedding 之外的新预测信号。作者据此把特征分成两类：对齐型特征表示'把 latent 信息翻译成人能懂的规则'，互补型特征表示'补上 embedding 的盲区'。
    - 通俗讲解：可以把 embedding 看成一个压缩过的黑盒用户画像，EAFD 做的事是同时问两个问题：这个新特征是不是在解释黑盒里已经有的内容？如果不是，它能不能带来额外效果？这样就不会再盲目堆特征，而是知道哪些特征是在'翻译模型'，哪些特征是在'补模型短板'。
    - 例子：比如输入是一串交易或行为事件，LLM 提出'最近30天活跃次数'这个特征。若它能被 embedding 很好重建，但加到下游模型里收益接近 0，说明它主要是在解释已有表征；再比如提出'金额序列的自相关'或'类别集中度 HHI'，如果 embedding 很难表达，但加入后 AUC 提升，就说明这是典型补盲特征。
  - **点 2：反思式迭代生成**
    - 技术细节：EAFD 不是一次性吐出一批特征，而是一个带反思反馈的 agent 循环。每轮里，LLM 读取原始序列数据和上轮反馈，反馈内容在可解释模式下包括各候选特征的对齐分数及正反案例，在性能模式下包括各特征的效用分数和下游模型在验证集上的重要性排序；随后 LLM 生成新的可执行代码来定义特征。若代码报错，系统会进入自纠错调试流程继续修复，成功执行后再做评估，并在固定特征预算下选择 uplift 最高的互补特征进入最终模型；论文实现里跑了 5 轮，用 CatBoost 评估生成特征。
    - 通俗讲解：这有点像一个会复盘的特征工程师：先出一版特征，看哪些只是重复、哪些真正有用，再根据反馈改下一版。关键不是让 LLM 随便想点子，而是把'效果反馈'喂回去，让它越来越贴近当前 embedding 的缺口。
    - 例子：第一轮如果系统发现大量'总消费金额''总次数'之类特征和 embedding 高度重复，下一轮就会倾向去探索'最近 7 天到 30 天的变化''首末行为间隔''周期波动'这类更细的时间结构。这样在同样的特征预算下，后续轮次更容易挑出真正增益大的特征，而不是重复造轮子。
  - **点 3：盲区诊断与增强**
    - 技术细节：论文不只把 EAFD 当特征生成器，还把它当 embedding 诊断工具。作者统计了 43 个发现特征，并按金额、类别、时间、活跃度分组，用对齐分数看不同编码器能解释哪些信息：CoLES 更擅长类别和活跃度，但时间维较弱；NTP 活跃度强，但金额和类别弱；LLM4ES 在时间上更强但活跃度较弱。基于这种盲区分析，作者直接修改 CoLES 的输入表示，在金额上加入对数、指数、分段线性编码，在时间上加入 Time2Vec，结果时间类可重建性提升很多，Rosbank 的下游 AUC 从 0.835 提到 0.845；此外他们还把这种解释信号用于隐私擦除，用去相关正则压制 embedding 中某些敏感属性。
    - 通俗讲解：这一步的价值是：EAFD 不只是告诉你'加什么特征能涨点'，还告诉你'现有序列模型到底哪里没学好'。有了这个诊断，团队可以反过来改 encoder，而不是长期靠补丁式特征堆起来。
    - 例子：如果分析发现某个序列 encoder 几乎不表达时间节奏，那么广告侧就可能在'近 3 天突然活跃''每周固定回访'这类模式上吃亏。此时既可以短期加时间特征补救，也可以长期把时间编码改进到基础模型里，让后续更多任务共享收益。
  - **点 4：实验显示稳健增益**
    - 技术细节：在四个公开金融事件序列数据集和一个大规模私有多目标数据集上，EAFD 都比 embedding-only 和多种自动特征工程基线更稳定。公开集上，相对最强预训练 embedding 的提升最高到 5.8%，对较弱 embedding 的提升最高到 19.3%；例如 Rosbank 上 CoLES 从 0.835 提到 0.872，DataFusion 上 CoLES 从 0.738 提到 0.781，NTP 在多个数据集上提升更大。论文还给出迭代过程示例，在 Rosbank 上验证 AUC 从 0.8206 持续涨到 0.8396，常见高价值特征包括首末交易间隔、最近窗口频次、类别集中度 HHI、指数加权移动平均和金额自相关等。
    - 通俗讲解：结果说明这套方法不是只对某一个模型或某一类任务有效，而是对强 embedding 能继续挖边际收益，对弱 embedding 更能大幅补课。对工业团队来说，这比'换一个更大模型'更现实，因为它兼容现有表征和现有树模型流水线。
    - 例子：如果一个广告账户当前已经有较强的用户序列 embedding，EAFD 可能只能再带来几个点以内的提升，但这些提升往往来自很具体的行为节奏特征，容易上线和排查。如果底层 embedding 还比较粗糙，EAFD 反而可能通过补足时序和分布统计带来更明显的收益。
- **对广告的启发：** 最适合层级：特征工程层与序列建模层之间的桥接层，最适合用于广告中的用户行为序列、转化路径、再营销人群、账户风控和出价前预估特征补强。；价值：广告里也长期存在同样矛盾：一边是序列 embedding 或大模型表征，一边是大量人工统计特征和规则特征。EAFD 的可迁移价值在于，它能围绕现有 embedding 自动发现'解释型特征'和'互补型特征'，特别适合补充近窗频次、行为间隔、类目集中度、消费或出价波动、周期性回流等模式。一个直观用法是：先冻结现有用户序列 encoder，再让系统围绕点击、曝光、加购、转化等事件自动生成候选统计特征，用联合模型筛出能提升 CTR、CVR 或 LTV 预估的少量特征，从而在不大改主干模型的前提下提升效果和可解释性。；风险：论文场景是金融交易，不是广告，属于强迁移而非直接证据，因此其收益幅度未必能原样迁移到广告。文中也没有充分展开候选特征空间、提示词设计、特征预算细节和线上延迟评估，所以在广告落地时，需要重点关注三类风险：一是 LLM 生成特征可能泄漏标签时间窗或引入未来信息；二是多轮代码生成和验证成本较高，不一定适合高频全量自动运行；三是广告数据比金融更受曝光机制和策略反馈影响，某些高效用特征可能只是学习了投放偏差而非用户真实兴趣，因此需要更严格的时间切分、因果校验和线上灰度。
