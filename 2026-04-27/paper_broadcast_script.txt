早上好，今天是四月二十七号。先给你一个数字感受一下今天的重点：在一种主流的生成式推荐强化学习训练里，采样出来的学习样本组，有百分之八十五是全零奖励，百分之十三只有一条命中，真正有结构价值的只有百分之二。

换句话说，模型每天烧着大把 GPU，其实九成八的算力都在学一堆没法学的东西。

今天这期的主判断就从这儿出发：当生成式模型开始接管推荐和广告的排序链路，真正的瓶颈已经不是模型多大、prompt 多巧，而是训练信号本身干不干净、学不学得动。今天这批论文里，有好几篇不同角度都在回答同一个问题——怎么把稀疏、噪声、超长的原始数据，重塑成模型真的能用的信号。

先给你三个值得记住的事件。

第一，ReCast 这篇论文直接改造生成式推荐的强化学习信号，用"先修复、再对比"的方式，把那些全零的废组救活，再只挑最关键的一正一负去更新。这是今天最值得广告同学看的一篇。

第二，一篇叫 SLIDERS 的长文档问答工作，换了一条完全不同的思路——不把证据塞回 LLM 上下文，而是抽到关系数据库里用 SQL 来算。对广告的离线知识库构建很有启发。

第三，有一篇叫 Rethinking Semantic Collaborative Integration 的论文，直接质疑了现在主流的做法——把 LLM 语义向量和协同过滤向量简单对齐。作者的判断是：这两种表示根本不是一个东西，硬对齐会把局部结构搞坏。这是在提醒所有做语义加协同融合的团队，默认范式可能本身就是错的。

好，先展开今天最重要的 ReCast。

先说它为什么重要。生成式推荐现在越来越流行用 GRPO 这类组内归一化的强化学习来训练，每次采样一组候选，比较组里的好坏来更新模型。听上去很合理，但在推荐这种单目标、稀疏命中的场景下，一组三十二个采样里经常一个都没命中，整组奖励全是零。模型看着一堆"全错的答案"，根本不知道该往哪个方向调。

ReCast 的第一个关键动作很朴素，也很聪明：既然全错，那就塞一个真实的正确答案进去。具体来说，它把组里"最不像目标"的那条样本踢掉，换成 ground-truth 作为锚点。相当于学生整张卷子做错了，老师直接把标准答案贴在旁边，让学生至少有个对照物。注意它不是随便替换，而是精准替换掉最没信息量的那条，这样那些"差一点点就对"的近似答案仍然被保留，因为那些才是模型真正要学会区分的难样本。

第二个关键动作叫边界对比。传统做法是把组里三十二条样本的奖励做归一化，梯度平均分给每一条。ReCast 说不用这么麻烦，只挑两条：最强的正样本，和最像目标但还是错的负样本。其他三十条梯度全置零。这样梯度方向更干净，模型会被迫去学会区分"几乎蒙对"和"真正答对"的那条细线。

这个设计还顺带带来一个额外好处——采样宽度和更新宽度解耦了。你可以继续广撒网采三十二条，但实际回传梯度只走两条。论文里 actor 更新速度直接快了十六倍，显存降了约百分之十六。

把这套思路搬到广告场景，你会发现它太对口了。广告里真正的转化、点击本来就稀疏，生成式召回和排序一旦用上 RL，全零组会比推荐还严重。ReCast 的修复思路可以把真实转化的广告注入作为锚点；边界对比则让模型专心去分辨"最像目标的误召回"和"真正的目标"，这对于相似创意、相似商品之间的精细区分是很对的方向。

当然也有风险。广告需要探索广度和创意多样性，如果只盯边界一对样本，长尾和冷启动可能会受影响。另外它里面那个判断"像不像目标"的结构相似度函数，是针对商品 ID 设计的，广告场景得重新定义。

接下来说第二篇重点，SLIDERS，也就是那篇长文档问答的工作。

它针对的问题是：金融、医疗这种场景里，分析师要跨几百份文档、上千万 token 去综合证据。再大的上下文窗口也装不下。传统做法是切块、抽取、再拼回去让模型读一遍——但抽出来的证据多了，本身又变成一个新的超长上下文。

SLIDERS 的思路是：干脆不让 LLM 再去读大段拼接证据了。它让 LLM 只做一件擅长的事——读一小块，抽出结构化事实；剩下的聚合、去重、比大小，全交给数据库和 SQL。每条抽出来的记录不光有取值，还带着原文引用和抽取理由，所以数据是可追溯、可审计的。

最有意思的是它的"对账"机制。同一个公司可能被抽出五条冲突的长期借款数据，有的写零，有的是明细，有的是合计。传统方法遇到这种只能硬拼，模型很容易蒙。SLIDERS 让一个 SQL Agent 按公司分组，看着每条记录的原文引用和理由，判断是该合并、该去重还是该解冲突，然后写 SQL 去改。因为每一步都是可审计的 SQL，而不是黑箱推理，整个流程稳得多。

对广告同学有什么用？最直接的迁移场景是离线知识库。商家资质审核、品牌素材合规、长尾类目知识、竞品财报解读——这些场景里都有大量非结构化长文档。可以借鉴 SLIDERS 的路子，把它们一次性抽成带溯源的关系表，后面无论是风控、定向还是创意生成，都能复用同一份结构化库，不用每次重新塞进 LLM。多渠道抽出来的商家属性冲突，也可以用对账思路合并。

但要注意，这套方法每题要几分钟、成本近一美元，完全不适合实时竞价链路，适合的是离线批处理。

最后再补一句今天值得顺手看的一个基础概念——表示学习。

今天这三篇论文，不管是 ReCast 改信号、SLIDERS 抽事实，还是那篇质疑语义协同对齐的论文，本质都在回答一个问题：我们到底要把原始数据压成什么样的表示，下游才学得动。表示学习不是简单把一段文本或一个用户压成向量，而是要让这个向量里真正留下影响任务的结构信息，把噪声压掉。很多论文表面在做召回、排序、生成，真正的改进都发生在表示层。理解这一点，你就能更快抓住一篇论文真正的创新在哪。

今天如果只带走一个判断，我希望是这个：当模型能力越来越强，训练数据和训练信号的质量反而成了新的天花板。不管是生成式推荐 RL 的稀疏奖励，还是长文档问答的证据聚合，甚至是语义和协同的融合，今天这批论文都在提醒我们——别急着堆模型，先问一句，你给它的信号，真的学得动吗？
