早上好，今天是 4 月 29 日。先给你一个有点反直觉的数字：有一篇论文在 RTB 广告拉活场景里做了个实验，把用户转化之后在站内消费的内容偷偷塞进训练数据，留存预测的 AUC 直接涨了一大截。但上线当然不能这么干，因为这就是典型的特征穿越。问题是，它说明了一件事——决定你出价准不准的，往往不是你手里的特征，而是你拿不到的那部分未来。今天这期，我们就从这个"拿不到的边界"讲起。

如果让我用一句话提炼今天这批论文的主判断，我会说：推荐和广告系统的下一个胜负手，正在从"更好的打分模型"往"更好的表示"迁移。不管是生成式重排、生成式召回排序一体化、还是今天这篇拉活留存建模，真正动的那一层，都是底下那张把用户、物品、行为、甚至未来信号映射成向量的"地图"。排序只是这张地图上画一条线，召回只是找一块近的区域，而谁的地图画得更准、更完整，谁的系统上限就更高。

今天值得你记住三件事。第一件，是刚才那篇讲 RTB 拉活留存的《Break the Inaccessible Boundary》，它用蒸馏的方式把"出价后才能看到的用户行为"偷渡回了竞价时刻。第二件，是 Co-Director，用一个分层的 agent 系统自动拍广告短片，还顺带提出了一个面向广告场景的评测基准。第三件，我们放在后面讲，是生成式重排的一个新思路，它把重排从"在候选里挑"改成了"在全局 item 空间里生成"。

先展开第一篇，今天最值得广告同学花时间的论文。

场景是这样的：一个用户已经卸载了你的 App，或者很久没回来了。广告系统要决定，要不要在 RTB 竞价里为这次曝光出价、出多少。判断的核心，是预测他回来之后会不会留下来，也就是 LT1、LT7、LT30 这些留存指标。问题在于，预测这件事最强的信号，恰恰是他回流之后刷了什么视频、停留在哪些页面——可这些内容，在你出价那一刻根本不存在。

作者做的事情，思路其实很干脆。他们先训一个"作弊版"的 teacher 模型，故意让它看未来，把用户回流后几天的内容序列喂进去，学会压缩成一个向量，去预测留存。然后再训一个 student，student 只能看出价时真正能拿到的东西——画像、历史行为、广告上下文，它的任务就是去猜那个作弊向量。上线的时候，teacher 丢掉，只留下 student。

关键的机制有两个。一个是 teacher 那边的层级编码：把回流后每一天的行为先压成一句"当天摘要"，再按时间顺序串起来，而且第三天不许偷看第四天，这样 teacher 内部都不会出现跨天泄露。这个设计让它每一天的表示都能对应到 LT1、LT7 这种按天定义的留存目标，teacher 本身就是结构化的。

另一个关键点在消融结果里。作者画了一条曲线：student 的表示跟 teacher 越像，留存 AUC 就越高，几乎是单调关系。这说明被迁移过去的，确实是"未来内容的潜在信息"，不是什么噪声对齐。线上 A/B 里，针对已卸载用户的再激活设备数涨了 34%，LT30 涨了 22%。

为什么这件事值得所有做广告的人记住？因为广告和站内推荐之间，一直有一道信息墙。推荐系统看得到用户落地之后的一切，而出价系统只能站在门外猜。这篇论文等于在说：你猜不到没关系，你可以训练的时候让一个模型先作弊，然后把它的"感觉"蒸馏回来。对拉活、唤醒、重定向这些决策早于消费的场景，这是一个可以直接迁移的范式——teacher 端换成站内真实行为，student 端换成曝光前可得特征，去提 LTV 预估、ROI 预估。

当然也有坑。最大的坑是 teacher 要先训好、先冻住，如果让 teacher 和 student 一起学，两边会一起崩。论文里没有冻 teacher 的那组消融，AUC 直接掉了七个点。第二个坑是归因，广告主能不能拿到深度站内行为本身就是个问题。第三是离线 AUC 到线上收益之间还隔着一层出价策略，别以为 AUC 涨了钱就一定来。

讲完这个，我们换一个画面，看第二篇广告强相关的论文，Co-Director。

如果说前一篇是在排序和出价的底层动刀，Co-Director 动的是创意层。现在大家用 LLM 串扩散模型拍广告片，典型做法是一步一步写 prompt，剧本一个 prompt、分镜一个 prompt、画面一个 prompt。问题是每步都有小错，错误会一层层放大，拍出来人物会换脸，产品会变样，整体还没有一致的创意方向。

Co-Director 做了两件事。第一件比较有意思：它把"创意方向"本身当成一个多臂老虎机的问题。具体是三根轴——卖点怎么讲、故事怎么讲、视觉什么风格，每根轴几个档位。系统拍一条片子，就请一个多模态大模型当评委，分别给这三根轴打分，然后分别更新对应档位的评分。下一次就更倾向于历史表现好、但偶尔也探索没试过的组合。

这个设计对做 DCO、做动态创意的人特别有启发。以前调创意基本靠人拍脑袋，现在你可以把创意维度拆成几条正交的轴，在线上让真实的 CTR、CVR 当奖励，让系统自己去找"哪种创意打哪类人群更有效"。换句话说，创意优化第一次变成了一个可以在线学习的问题。

第二件事是分层的 agent 协作。关键不是 agent 多，而是所有子 agent 的 prompt 都从同一份"创意配置"被改写出来，剧本、分镜、画面、音乐不会打架。还有一个工程细节值得抄：关键帧先生成、先钉死人物长相和产品外观，然后再让视频模型只负责让它动起来。这个顺序一换，身份漂移的问题就缓解了一大半。消融里显示，如果把关键帧那层的序列级审核去掉，视觉保真度直接掉十分。

风险也要提一句。MLLM 当评委判断"营销吸引力"，很容易优化出"AI 喜欢但用户不买单"的风格。真正落地时，这个奖励信号必须换成真实转化，不然就是自嗨。

好，第三件值得提一下的事，是《From Local Indices to Global Identifiers》这篇生成式重排。它指出了一个很隐蔽的问题：以前的 list-wise 重排，输出层的第三个神经元在这次样本里代表商品 A，下次样本里可能代表商品 B，模型其实没办法对商品本身建立稳定的理解。它的做法是把每个 item 表示成一串离散 token，重排变成一个生成任务，直接在全局 item 空间里自回归地"写"出序列。

为什么顺带提一句？因为这和今天我们说的主判断是一条线的——它动的还是底下那张地图。只不过 OneRec 系列把召回排序统一在生成式框架里，这篇把重排也拉进来了。如果你在做广告重排，这个方向值得盯着，因为它理论上能让重排跳出召回给的小盒子。

最后给你留一个判断。今天这批论文放一起看，会发现一个共同点：大家都在想办法把"系统本来看不到的信息"，用某种方式塞进决策的那一刻。拉活那篇塞的是未来行为，Co-Director 塞的是创意空间里没探索过的组合，生成式重排塞的是召回之外的候选。竞争的焦点，正在从"用好手里的特征"变成"合法地扩大你能用的信息边界"。

谁能把这条边界往外推一点，谁就能在下一轮出价、下一轮曝光、下一次创意里多赢一点。今天就到这里，祝你有个判断清晰的一天。
